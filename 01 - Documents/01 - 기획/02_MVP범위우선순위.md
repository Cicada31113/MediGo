# 01_02 · MVP 범위 & 우선순위 – MediGo

> 문제 1 · "지금 반드시 할 것"과 "나중에 할 것"을 구분

## 1. MoSCoW 우선순위 표

### 1.1 Must Have
- M-01: 잠금/홈 화면 + 알림 진입 플로우
- M-02: 약국 검색/리스트/지도 화면
- M-03: 약국 목록/검색/재고확인 API
- M-04: 주문 생성 + 배달 현황(애니메이션 포함)
- M-05: 로컬 SQLite 저장 및 재실행 시 데이터 유지

### 1.2 Should Have
- S-01: 무설치 데스크톱 exe 패키지
- S-02: 처방전 이미지 썸네일 및 업로드 API
- S-03: ETA(예상 도착 시간) 계산 및 표시

### 1.3 Could Have
- C-01: 다국어 지원(영문 UI)
- C-02: 복약 알림 푸시(후속 릴리즈)

### 1.4 Won't Have (이번 범위 제외)
- W-01: 실제 결제/보험 청구 연동
- W-02: 병원/약국 EMR 연동
- W-03: 실시간 GPS 기반 라이더 추적

## 2. RICE 스코어(선택)
| 기능 | Reach(도달) | Impact(임팩트) | Confidence(확신) | Effort(공수) | RICE 점수 |
|------|------------|----------------|------------------|--------------|-----------|
| 잠금/홈 화면 플로우 | 100 | 높음 | 높음 | 3 | 33.3 |
| 약국 검색/리스트/지도 | 100 | 매우 높음 | 높음 | 4 | 25.0 |
| 주문 생성 + 배달 현황 | 100 | 매우 높음 | 중간 | 5 | 20.0 |
| 처방전 업로드 | 60 | 중간 | 중간 | 3 | 12.0 |

## 3. MVP 범위 요약 표
| 영역 | 포함 기능 (IN) | 제외 기능 (OUT) | 비고 |
|------|----------------|------------------|------|
| 사용자앱 | 알림→앱 진입, 처방전 보기, 약국 검색·선택, 주문 생성·현황 보기 | 결제, 로그인/회원관리 | 해커톤 데모 기준 |
| 관리자 | (이번 범위 없음) | 주문 대시보드, 통계, 고객관리 | 후속 릴리즈 |

## 4. 일정 기반 스코프 슬라이싱
### 4.1 현재 텀프로젝트 기간 내 구현
- 1주차: 문제 정의/PRD, 기본 UI 와이어프레임, API 설계  
- 2주차: 약국/주문 DB 모델 + 약국 목록/검색 API, 기본 지도 화면  
- 3주차: 주문 생성/배달 현황 애니메이션, 로컬 DB 영속성  
- 4주차: 무설치 exe 빌드, 테스트/버그 픽스, 문서/발표 정리

### 4.2 후속 릴리즈로 밀어낼 항목
- 결제·실제 약국 시스템 연동: 법/보안 검토 및 파트너십 의존성이 크므로 본 프로젝트 이후 별도 트랙으로 진행  
- AI 기반 복약 지도/챗봇: 모델 선택·튜닝·검증에 시간이 많이 필요하므로 POC 이후 단계로 이관
