# 03_05 · 기술 요약 – MediGo

> 문제 3 · 아키텍처 · API/데이터 · 로깅 요약

## 1. 아키텍처(1장 요약)
- 클라이언트: 정적 웹(Leaflet, Pretendard) + 데스크톱(webview, PyInstaller onefile)  
- 서버: FastAPI + Uvicorn (단일 프로세스, `/`로 정적 파일 서빙 + `/api/*`)  
- 데이터: SQLite (로컬, `~/.medigo/db.sqlite`), 업로드 파일 `~/.medigo/uploads/`

## 2. 핵심 API / 데이터 모델(1장 요약)
- API: `/api/pharmacies`, `/api/orders`, `/api/prescriptions` 등 (자세한 내용은 02_02 참조)
- 데이터 모델: `Pharmacy`, `Order`, `Prescription` (models.py 참조)

## 3. 로그/모니터링 포인트(1장 요약)
- 주요 이벤트 로그:
  - 주문 생성/상태 변경
  - 예외 발생 로그
- 헬스 체크: `/health`
