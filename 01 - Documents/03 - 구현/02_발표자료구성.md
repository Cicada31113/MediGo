# 03_02 · 발표 자료 구조 – MediGo

> 문제 3 · 발표 구조 가이드

## 1. 문제
- 현실의 처방전/약 배달 불편 요약  
  - "병원은 끝났는데, 약 받으러 또 줄 서야 한다"  
  - "가봤더니 약이 없어서 다른 약국으로…"  
- 사용자/상황/페인포인트  
  - 직장인·보호자·고령자가 공통적으로 느끼는 이동/대기 스트레스

## 2. 해결책(MVP)
- MediGo 한 장으로 요약 (한 슬라이드)
- 핵심 가치: "병원 앞에서 기다리지 않고 집에서 약을 받는다"

## 3. 데모 시나리오(와우 모먼트)
- 90~120초 데모 흐름:
  - 알림 → 앱 진입 → 처방전 확인 → 약국 선택 → 배달 현황

## 4. 성과/KPI 가설
- 첫 사용자의 플로우 완료율 ≥ 80%  
- "다음에도 쓰고 싶다"고 답한 비율 ≥ 80%  
- 약 수령까지 추정 시간 30분 이상 단축(기존 경험 대비 체감 기준)

## 5. 기술/운영
- 아키텍처 다이어그램 1장
- 운영/모니터링 요약

## 6. 로드맵
- 다음 단계 기능 (실제 약국/병원 시스템 연동 등)
